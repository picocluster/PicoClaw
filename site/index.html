What is PicoCluster Claw?

A two-node AI cluster that fits on your desk. Run local LLMs with GPU acceleration, chat through a browser UI, and let the AI control LEDs, read sensors, manage files, and more — all on your own hardware.

🧠
RPi5 — clusterclaw

Agent Orchestrator

Raspberry Pi 5 (8/16GB) runs the web UIs, MCP tool servers, LED status daemon, and Docker containers for ThreadWeaver, OpenClaw, and Caddy HTTPS. The brain that coordinates everything.

🚀
Jetson — clustercrush

GPU Inference Engine

NVIDIA Jetson Orin Nano Super (8GB) runs Ollama with CUDA acceleration. 9 models installed, hot-swappable. 8GB unified memory handles up to 8B-parameter models at full speed.

💡
Blinkt! LED Strip

Physical AI Feedback

8 APA102 LEDs on the GPIO header show cluster state at a glance. Teal scanner when idle, amber during model loading, green on success, red on errors, cyan chase during inference. The AI can control them via MCP tools.

Two nodes, one LAN, zero cloud. Everything runs locally.

Software Stack

Three AI interfaces, one unified tool layer.

Chat Interface

ThreadWeaver

A chat-first UI with branching conversations, search, notebooks, and multi-model switching. Supports 5 LLM providers: local (Ollama), Anthropic (Claude), OpenAI (GPT), Google (Gemini), and xAI (Grok).

  • Branch any conversation at any message
  • Switch models mid-conversation
  • 28 MCP tools available to every provider
  • Tools toggle: OFF by default for fast local chat, ON when you need them
  • Vision support with llava:7b and moondream:1.8b
Agent Framework

OpenClaw

An autonomous agent that plans, executes tools, browses the web, and reports back. Includes a Canvas dashboard (HTTPS), terminal TUI, and messaging channel integrations.

  • Goal-oriented autonomy — give it a task, it figures out the steps
  • Browser automation via CDP relay
  • Telegram, Discord, Slack, WhatsApp, and 15+ more channels
  • LED bridge: purple thinking, cyan tool use, green success, red error
  • HTTPS dashboard via Caddy + SSH tunnel
State Management

AgentStateGraph

An AI-native versioned state store compiled natively on the Jetson. Git-like branching and speculations for agent state, with intent metadata on every commit. Optional — connect via pc-asg connect.

  • 19 additional MCP tools (branch, merge, speculate, diff, blame, epochs)
  • SQLite storage, persists in ThreadWeaver sandbox
  • Sealed epochs for tamper-evident audit trails
  • BSL-1.1 licensed, open-core
Control Panel

Portal

A web-based control panel at http://clusterclaw/ with everything a user needs — no SSH required for basic operations.

  • Live status indicators (ThreadWeaver, OpenClaw, Ollama)
  • Cluster restart / shutdown buttons
  • LED color picker, pulse effects, progress slider
  • 28 copyable example prompts organized by MCP server
  • 20+ expandable FAQ entries
  • SSH tunnel instructions, installed models table, hardware specs

AgentStateGraph

An AI-native versioned state store — compiled natively on the Jetson Orin Nano. Git-like branching for agent state with intent metadata on every commit.

19 Additional MCP Tools

Connect AgentStateGraph as an opt-in MCP server and the LLM gets branching, merging, speculations, structured diffs, blame, epochs, and agent sessions.

  • agentstategraph_get / set / delete — state CRUD with intent metadata
  • agentstategraph_branch / merge — namespace-aware branching
  • agentstategraph_speculate — lightweight what-if branches (O(1) creation)
  • agentstategraph_diff / blame / query — structured history
  • agentstategraph_seal_epoch — tamper-evident audit trails

pc-asg connect to enable, pc-asg disconnect to remove. State persists in SQLite.

Why It Matters

Traditional databases don't capture why a change was made, who the agent was, what it was trying to accomplish, or how confident it was. AgentStateGraph makes provenance a first-class primitive.

  • Every commit carries intent category, description, reasoning, and confidence
  • Speculations let agents explore approaches before committing
  • Sealed epochs provide cryptographic proof of what agents did
  • Query by agent, intent, tags, confidence range, date range

Privacy & Security

Your data, your hardware, your rules.

🔒

Local-First Inference

The local provider runs entirely on your LAN. Model weights on the Jetson's NVMe, conversations on the RPi5's SD card. Nothing leaves your network unless you explicitly switch to an external provider.

🛡️

Localhost-Only Ports

ThreadWeaver (5173, 8000) and OpenClaw (18789) are bound to 127.0.0.1. LAN clients must SSH-tunnel to Caddy HTTPS (5174, 18790). The portal (port 80) is the only public-facing web service.

🔥

Firewall + Ollama Isolation

Ollama on clustercrush only accepts connections from clusterclaw's IP. Direct LAN access is blocked by ufw. No authentication because there's no network path to exploit.

📦

Sandboxed File Access

The LLM's file tools are restricted to a Docker-volume sandbox. Path traversal attempts are rejected at the Python level. The model cannot read host files, container internals, or your computer's filesystem.

Getting Started

From unboxing to first chat in under 5 minutes.

1️⃣

Plug In & Boot

Connect power to both nodes. The LED strip goes teal (booting), then amber (model loading), then green (ready). Takes about 2 minutes.

2️⃣

Add Hostnames

Add to your computer's /etc/hosts:
10.1.10.220 clusterclaw
10.1.10.221 clustercrush

3️⃣

Open the Portal

Browse to http://clusterclaw/ for the control panel. Status, LED control, example prompts, FAQ — everything is there.

4️⃣

Start Chatting

Run the SSH tunnel:
ssh -L 5174:localhost:5174 -L 18790:localhost:18790 picocluster@clusterclaw
Then open https://localhost:5174 and start talking to your local LLM.

Frequently Asked Questions

Highlights from the full 26-question FAQ.

Can external LLMs (Claude, GPT-4) use the local MCP tools?
Yes, fully. The external LLM sees tool schemas and decides when to call them. ThreadWeaver executes the call locally on clusterclaw. The external model never touches your hardware directly — it describes what it wants, and your cluster acts as the trusted executor. All 28 tools work with every provider.
What gets sent to external providers?
Your prompts, the LLM's replies, tool schemas, and tool call results. NOT sent: your API keys beyond auth, files outside the sandbox, other conversations, raw local inference. For strict privacy, use the local provider — everything stays on your LAN.
Why do some models not support tool calling?
Tool calling requires structured JSON output that not all models were trained for. ThreadWeaver auto-detects this and falls back to chat-only with a clear notice. Models like gemma3:4b, llava:7b, and starcoder2:3b work fine for plain chat and vision — they just can't invoke MCP tools.
Why is the default model llama3.1:8b instead of the faster 3b?
llama3.2:3b handles 1-2 tool calls fine but degrades on multi-turn conversations — it starts printing tool calls as plain text instead of making real structured calls. llama3.1:8b is slower per reply but reliably chains 3+ tool calls. With tools OFF (the default for local), you can switch to 3b for faster plain chat.
Can the LLM read files on my computer?
No. The files MCP server is sandboxed to /tmp/clusterclaw-sandbox inside the ThreadWeaver container. Path traversal is rejected at the Python level. Your computer's filesystem is never exposed.
What do the LED colors mean?
Teal scanner: idle. Amber: model loading. Green burst: success. Red flash: error. Purple pulse: agent thinking. Cyan chase: active inference. Blue fade: shutting down. The LLM can also control them directly: "Make the LEDs purple."
How do I add more models?
SSH to clustercrush and run pc-pull <model-name>. Browse ollama.com/library for tags. Or ask the LLM: "Pull the qwen3 model from Ollama."
What is AgentStateGraph?
A Rust-native versioned state store for AI agents. Think "git for agent state" with intent metadata, speculations (lightweight what-if branches), and sealed epochs for audit trails. It's compiled on the Jetson and available as an opt-in MCP server with 19 tools via pc-asg connect.
